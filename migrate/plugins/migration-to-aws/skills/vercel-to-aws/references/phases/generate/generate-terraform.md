# Generate Phase: Core Terraform Configuration

> Self-contained fragment. Always runs. Produces the core Terraform files that
> every Vercel-to-AWS migration needs regardless of outcome: provider config,
> variables, outputs, VPC, security groups/IAM, gitignore, and tfvars example.

**Execute ALL steps in order. Do not skip or optimize.**

---

## Step 1: Generate `terraform/main.tf`

```hcl
# Vercel-to-AWS Migration — Terraform Configuration
#
# Generated by the vercel-to-aws migration skill.
# This configuration implements the architecture recommended in recommendation.json.
#
# IMPORTANT: This Terraform is aligned with the BALANCED cost scenario from
# estimation-infra.json. Premium/Balanced/Optimized are three pricing SCENARIOS
# for the same architecture — not three separate Terraform stacks. To realize
# Premium (higher HA) or Optimized (cost-minimized) postures, edit instance
# classes, Multi-AZ settings, and capacity reservations in variables.tf.
#
# Apply sequence:
#   1. terraform init
#   2. terraform plan -out=tfplan
#   3. terraform apply tfplan

terraform {
  required_version = ">= 1.5.0"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.80"
    }
  }

  backend "s3" {
    # Bootstrap: these resources are created by baseline.tf.
    # First run: terraform init -backend=false && terraform apply \
    #   -target=aws_s3_bucket.tfstate \
    #   -target=aws_s3_bucket_versioning.tfstate \
    #   -target=aws_s3_bucket_server_side_encryption_configuration.tfstate \
    #   -target=aws_s3_bucket_public_access_block.tfstate \
    #   -target=aws_dynamodb_table.tfstate_lock
    # Then re-run: terraform init  (migrates local state to S3)
    bucket         = "<project_name>-<environment>-tfstate-<account_id>"
    key            = "migration/terraform.tfstate"
    region         = "<aws_region>"
    dynamodb_table = "<project_name>-<environment>-tfstate-lock"
    encrypt        = true
  }
}

provider "aws" {
  region = var.aws_region

  default_tags {
    tags = {
      Project     = var.project_name
      Environment = var.environment
      ManagedBy   = "terraform"
      MigrationId = var.migration_id
      Source      = "vercel-to-aws"
    }
  }
}

data "aws_caller_identity" "current" {}
data "aws_region" "current" {}
data "aws_availability_zones" "available" {
  state = "available"
}
```

**Customization rules:**

- Substitute actual values from `clarify-answers.json` and `.phase-status.json`
  into the backend block's TODO placeholders as HCL comments (the actual values
  go into `terraform.tfvars.example`)
- The backend block uses TODO-style placeholders because backend blocks cannot
  reference variables — document the 2-step bootstrap in `terraform/README.md`

## Step 2: Generate `terraform/variables.tf`

**Always include these global variables:**

```hcl
variable "aws_region" {
  description = "AWS region for all resources"
  type        = string
  default     = "us-east-1"
}

variable "project_name" {
  description = "Project name used for resource naming and tagging"
  type        = string
}

variable "environment" {
  description = "Environment name (e.g., production, staging)"
  type        = string
  default     = "production"
}

variable "migration_id" {
  description = "Migration run identifier for traceability"
  type        = string
}
```

**Per-service variables** — Add based on which compute/peripheral fragments
will fire (read `recommendation.json.outcome` and `discovery.json.peripherals`):

- **Fargate (Outcome B):** `container_cpu`, `container_memory`,
  `desired_count`, `container_image`
- **Lambda (Outcome A/C-A):** `lambda_memory_size`, `lambda_timeout`
- **Database (Postgres peripheral):** `db_instance_class`, `db_storage_gb`,
  `db_engine_version`, `db_multi_az`
- **Cache (KV peripheral):** `cache_node_type`
- **Network:** `vpc_cidr` (default `"10.0.0.0/16"`)

Use sizing values from `estimation-infra.json.projected_costs.breakdown` as
defaults. Add a source comment on each:

```hcl
variable "db_instance_class" {
  description = "RDS instance class for PostgreSQL"
  type        = string
  default     = "db.t4g.small"
  # Sized from: estimation-infra.json (Balanced tier, Vercel Postgres 1-10 GB)
}
```

## Step 3: Generate `terraform/outputs.tf`

```hcl
output "migration_summary" {
  description = "Summary of migrated Vercel resources"
  value = {
    source_platform   = "vercel"
    target_region     = var.aws_region
    migration_id      = var.migration_id
    recommendation    = "<outcome from recommendation.json>"
  }
}
```

Add per-service outputs for connection information based on what compute and
peripheral fragments produce. Mark connection strings as `sensitive = true`:

```hcl
output "cloudfront_domain" {
  description = "CloudFront distribution domain name"
  value       = aws_cloudfront_distribution.main.domain_name
}

output "rds_endpoint" {
  description = "RDS PostgreSQL endpoint"
  value       = aws_db_instance.postgres.endpoint
  sensitive   = true
}
```

Only emit outputs for services actually generated. Use conditional blocks or
simply omit outputs for services not in the design.

## Step 4: Generate `terraform/vpc.tf`

Always emit a new VPC — Vercel has no VPC equivalent, so there is never an
"existing VPC" path (unlike Heroku's Private Space peering detection).

```hcl
# VPC — New VPC for Vercel-to-AWS migration
# Vercel has no VPC equivalent; all AWS compute needs a VPC.

resource "aws_vpc" "main" {
  cidr_block           = var.vpc_cidr
  enable_dns_support   = true
  enable_dns_hostnames = true

  tags = {
    Name = "${var.project_name}-${var.environment}-vpc"
  }
}

# Public subnets (for ALB, CloudFront origin, NAT Gateway)
resource "aws_subnet" "public" {
  count                   = 2
  vpc_id                  = aws_vpc.main.id
  cidr_block              = cidrsubnet(var.vpc_cidr, 8, count.index)
  availability_zone       = data.aws_availability_zones.available.names[count.index]
  map_public_ip_on_launch = true

  tags = {
    Name = "${var.project_name}-${var.environment}-public-${count.index + 1}"
    Tier = "public"
  }
}

# Private subnets (for Fargate, Lambda, RDS, ElastiCache)
resource "aws_subnet" "private" {
  count             = 2
  vpc_id            = aws_vpc.main.id
  cidr_block        = cidrsubnet(var.vpc_cidr, 8, count.index + 10)
  availability_zone = data.aws_availability_zones.available.names[count.index]

  tags = {
    Name = "${var.project_name}-${var.environment}-private-${count.index + 1}"
    Tier = "private"
  }
}

# Internet Gateway
resource "aws_internet_gateway" "main" {
  vpc_id = aws_vpc.main.id

  tags = {
    Name = "${var.project_name}-${var.environment}-igw"
  }
}

# NAT Gateway (single — Balanced tier; Premium uses one per AZ)
resource "aws_eip" "nat" {
  domain = "vpc"

  tags = {
    Name = "${var.project_name}-${var.environment}-nat-eip"
  }
}

resource "aws_nat_gateway" "main" {
  allocation_id = aws_eip.nat.id
  subnet_id     = aws_subnet.public[0].id

  tags = {
    Name = "${var.project_name}-${var.environment}-nat"
  }

  depends_on = [aws_internet_gateway.main]
}

# Route Tables
resource "aws_route_table" "public" {
  vpc_id = aws_vpc.main.id

  route {
    cidr_block = "0.0.0.0/0"
    gateway_id = aws_internet_gateway.main.id
  }

  tags = {
    Name = "${var.project_name}-${var.environment}-public-rt"
  }
}

resource "aws_route_table" "private" {
  vpc_id = aws_vpc.main.id

  route {
    cidr_block     = "0.0.0.0/0"
    nat_gateway_id = aws_nat_gateway.main.id
  }

  tags = {
    Name = "${var.project_name}-${var.environment}-private-rt"
  }
}

resource "aws_route_table_association" "public" {
  count          = 2
  subnet_id      = aws_subnet.public[count.index].id
  route_table_id = aws_route_table.public.id
}

resource "aws_route_table_association" "private" {
  count          = 2
  subnet_id      = aws_subnet.private[count.index].id
  route_table_id = aws_route_table.private.id
}
```

## Step 5: Generate `terraform/security.tf`

Emit IAM roles and security groups scoped to the specific services generated.
The exact content depends on which compute fragment fires:

### Always-On Security Groups

```hcl
# Security group for ALB (public-facing)
resource "aws_security_group" "alb" {
  name_prefix = "${var.project_name}-alb-"
  vpc_id      = aws_vpc.main.id

  ingress {
    from_port   = 443
    to_port     = 443
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
    description = "HTTPS from internet"
  }

  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
    description = "HTTP from internet (redirect to HTTPS)"
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
    description = "All outbound"
  }

  tags = {
    Name = "${var.project_name}-${var.environment}-alb-sg"
  }

  lifecycle {
    create_before_destroy = true
  }
}
```

### Conditional Security Groups and IAM

- **Fargate (Outcome B):** Add `aws_security_group.ecs_tasks` (ingress from
  ALB SG only on container port), `aws_iam_role.ecs_task_execution` with
  `AmazonECSTaskExecutionRolePolicy`, `aws_iam_role.ecs_task` (minimal
  permissions for the app)
- **Lambda (Outcome A/C-A):** Add `aws_security_group.lambda` (egress-only),
  `aws_iam_role.lambda_execution` with `AWSLambdaVPCAccessExecutionRole`
- **RDS (Postgres peripheral):** Add `aws_security_group.database` (ingress
  from compute SG only on 5432)
- **ElastiCache (KV peripheral):** Add `aws_security_group.cache` (ingress
  from compute SG only on 6379)

All security groups follow least-privilege: ingress only from the specific
source that needs it, on the specific port required.

## Error Handling

| Error Category                                 | Behavior                                                                           |
| ---------------------------------------------- | ---------------------------------------------------------------------------------- |
| `estimation-infra.json` unavailable for sizing | Use conservative defaults (db.t4g.small, 512 CPU / 1024 MB Fargate, 128 MB Lambda) |
| Outcome is unresolved tiebreak `["A", "B"]`    | Surface to the generate orchestrator — it asks the founder before dispatching      |

## Scope Boundary

**This fragment covers core Terraform files (main, variables, outputs, VPC,
security, gitignore, tfvars.example) ONLY.**

FORBIDDEN — Do NOT include ANY of:

- `baseline.tf` (that's `generate-baseline.md`'s job)
- Compute-specific resources like ECS, Lambda, or ALB (those are the compute
  fragments' jobs)
- Peripheral resources like RDS, ElastiCache, S3 (that's
  `generate-peripherals.md`'s job)
- Migration scripts or documentation

**Your ONLY job: emit the core Terraform skeleton. Nothing else.**
