== Why Migrate from OpenAI or Gemini to Amazon Bedrock? ==

|=Factor|=OpenAI / Gemini|=Amazon Bedrock
|**Model choice**|Locked to one provider|Claude, Llama, Nova, Titan, Mistral — switch models without code changes
|**Data privacy**|Data leaves your environment|VPC endpoints, no data used for training
|**Cost at scale**|Per-token, no volume discounts|Provisioned Throughput, Batch API, committed use discounts
|**Integration**|Separate auth, separate billing|IAM-native, consolidated AWS billing, CloudWatch metrics
|**Availability**|Single provider risk|Multi-model fallback (Claude → Nova → Llama)
|**Fine-tuning**|Limited control|Custom models, continued pre-training, knowledge bases

== Cost Comparison: OpenAI vs. Bedrock ==

=== Per-Token Pricing (On-Demand) ===

|=Model Tier|=OpenAI (per 1M tokens)|=Bedrock Equivalent (per 1M tokens)|=Savings
|**Flagship** (GPT-4o / Claude Sonnet)|$2.50 input / $10.00 output|$3.00 input / $15.00 output|Varies by volume
|**Fast** (GPT-4o mini / Claude Haiku)|$0.15 input / $0.60 output|$0.25 input / $1.25 output|Batch API 50% off
|**Reasoning** (o3 / Sonnet extended)|$10.00 input / $40.00 output|$3.00 input / $15.00 output|~60-70% savings

=== Cost Optimization on Bedrock ===

* **Batch API** — 50% discount for async workloads (classification, extraction, bulk processing)
* **Provisioned Throughput** — Reserved capacity at lower per-token cost for steady workloads
* **Model selection** — Route simple tasks to Haiku, complex to Sonnet (the plugin auto-detects this pattern)
* **Caching** — Prompt caching for repeated system prompts (up to 90% savings on cached tokens)

**To get a precise cost comparison:** Upload your OpenAI usage CSV or GCP BigQuery billing export and the plugin calculates your exact monthly cost on Bedrock.

== Code Migration: OpenAI SDK → Bedrock SDK ==

=== Python — Chat Completions ===

{{code language="python"}}
# ❌ Before: OpenAI
from openai import OpenAI
client = OpenAI(api_key="sk-...")

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Explain quantum computing"}
    ],
    temperature=0.7,
    max_tokens=1000
)
answer = response.choices[0].message.content
{{/code}}

{{code language="python"}}
# ✅ After: Amazon Bedrock (Converse API)
import boto3

client = boto3.client("bedrock-runtime", region_name="us-east-1")

response = client.converse(
    modelId="anthropic.claude-sonnet-4-20250514-v1:0",
    system=[{"text": "You are a helpful assistant."}],
    messages=[
        {"role": "user", "content": [{"text": "Explain quantum computing"}]}
    ],
    inferenceConfig={"temperature": 0.7, "maxTokens": 1000}
)
answer = response["output"]["message"]["content"][0]["text"]
{{/code}}

=== Python — Streaming ===

{{code language="python"}}
# ❌ Before: OpenAI streaming
stream = client.chat.completions.create(
    model="gpt-4o",
    messages=messages,
    stream=True
)
for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")
{{/code}}

{{code language="python"}}
# ✅ After: Bedrock streaming
response = client.converse_stream(
    modelId="anthropic.claude-sonnet-4-20250514-v1:0",
    messages=messages
)
for event in response["stream"]:
    if "contentBlockDelta" in event:
        print(event["contentBlockDelta"]["delta"]["text"], end="")
{{/code}}

=== Python — Embeddings ===

{{code language="python"}}
# ❌ Before: OpenAI embeddings
response = client.embeddings.create(
    model="text-embedding-ada-002",
    input="Hello world"
)
vector = response.data[0].embedding
{{/code}}

{{code language="python"}}
# ✅ After: Bedrock Titan Embeddings
import json

response = client.invoke_model(
    modelId="amazon.titan-embed-text-v2:0",
    body=json.dumps({"inputText": "Hello world"})
)
vector = json.loads(response["body"].read())["embedding"]
{{/code}}

== Migrating Agentic Frameworks to Bedrock ==

If you're using agentic frameworks, the migration is often a config change:

|=Framework|=OpenAI → Bedrock Change
|**LangChain**|Swap {{code}}ChatOpenAI(){{/code}} → {{code}}ChatBedrock(model_id="anthropic.claude-sonnet-4-20250514-v1:0"){{/code}}
|**LangGraph**|Same — underlying LLM provider swap only
|**CrewAI**|Set {{code}}llm_config{{/code}} to Bedrock provider
|**AutoGen**|Configure Bedrock endpoint in model config
|**OpenAI Agents SDK**|Replace with Bedrock Agents or Strands Agents SDK

The AWS Startup Advisor plugin detects these frameworks in your codebase and generates framework-specific migration code.

== FAQ ==

**Q: Can I migrate AI workloads without moving my infrastructure from GCP?**
A: Yes. The AI migration works standalone — you can keep your infrastructure on GCP and just switch your LLM provider to Bedrock.

**Q: What about fine-tuned OpenAI models?**
A: Bedrock supports custom model fine-tuning. The migration plan includes guidance on retraining with your dataset on Claude or other Bedrock models.

**Q: How do I handle OpenAI function calling / tool use?**
A: Bedrock's Converse API supports native tool use with the same pattern. The plugin generates the converted tool schemas automatically.

**Q: What about rate limits?**
A: Bedrock offers Provisioned Throughput for guaranteed capacity. No surprise rate limits during peak usage.

**Q: Can I use the OpenAI-compatible endpoint on Bedrock?**
A: Yes — Bedrock offers an OpenAI-compatible API endpoint, meaning some applications require zero code changes. The plugin identifies when this shortcut applies.
